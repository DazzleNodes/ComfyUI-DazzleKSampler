# Human Test Checklist: Explicit `latent_role` Widget (Issue #11)

**Target version:** v0.1.2-alpha
**Ships with commit:** TBD (`feat: explicit latent_role widget on Dazzle sampler nodes`)
**Companion automated tests:** 35 tests in `tests/test_latent_noise_protocol.py` (5×4 role × shape matrix + protocol invariants + legacy seed=-2 parity).

## What this is

A human-runnable companion to the unit tests for the new `latent_role`
widget. The unit tests exercise the pure helper `resolve_latent_as_noise()`
with mock latent dicts; this checklist exercises the full pipeline —
SmartResCalc → DazzleKSampler → VAE Decode → Preview — and the JS-side
seed-widget hiding behavior.

The widget replaces the legacy `seed = -2` magic with five explicit
choices: `auto`, `noise`, `latent_image`, `noise+latent_image`,
`seed_driven`. The `auto` default infers dispatch from the dict shape
regardless of the seed value. The `seed = -2` path still works under
`auto` (with a deprecation log) but will lose its special meaning in a
future release.

Run this before tagging `v0.1.2-alpha`.

## Prerequisites

1. **Working ComfyUI install** with `dazzle-ksampler` and
   `smart-resolution-calc` custom nodes loaded.
2. **A checkpoint** for sampling (any SDXL/Qwen/Wan/Flux works — the
   widget is model-agnostic).
3. **A test image** (any 512×768 or 1024×1024 portrait) for the img2img
   paths.
4. **Console access** — keep ComfyUI's terminal visible. The dispatch
   site logs the active role + detected shape on the first step:
   ```
   [DazzleKSampler] latent_role=auto, detected=shape_b, applied=shape_b, noise_seed=42
   [DazzleKSampler] WARNING: ...   (mismatch / informational)
   [DazzleKSampler] DEPRECATION: ... (only on seed=-2 under auto)
   ```
5. **Reset state between sections:**
   - Right-click each Dazzle sampler node and confirm `latent_role` is
     visible at top of the widgets stack.
   - Set `latent_role = auto` between most tests; switch explicitly per
     section.
   - For seed-hiding tests, observe the seed widget's appearance/disappearance.

## High-Value Verification (~5 minutes)

If you only have 5 minutes, run these.

### HV.1 — `auto` + Shape B dispatches without `seed=-2`

Validates: the modern dispatch fires on dict shape alone (no `-2` needed).

- [ ] **Steps:**
  1. Build: `SmartResCalc(image_purpose='dimensions only', fill_type='DazNoise: Brown', blend_strength=0.8) → DazzleKSampler → VAE Decode → Preview`. No image attached upstream.
  2. DazzleKSampler: `latent_role = auto`, `seed = 42` (any non-`-2` value).
  3. Queue.
- [ ] **Expected:**
  - Output reflects the upstream noise character (varies if you change `fill_type`).
  - Console:
    ```
    [DazzleKSampler] latent_role=auto, detected=shape_b, applied=shape_b, noise_seed=42
    [DazzleKSampler] WARNING: auto-mode dispatched to Shape B ... Pre-v0.1.2-alpha this required noise_seed=-2; ...
    ```
- [ ] **Why this matters:** The whole point of v0.1.2 is decoupling shape detection from the `-2` seed magic. The informational notice (Option C semantics) tells users they're on the modern path.

### HV.2 — `auto` + `seed=-2` still works, prints deprecation log

Validates: backward compat with v0.1.1 workflows.

- [ ] **Steps:**
  1. Same workflow as HV.1.
  2. Set DazzleKSampler `seed = -2`.
  3. Queue.
- [ ] **Expected:**
  - Output equivalent to HV.1 (Shape B dispatch).
  - Console includes:
    ```
    [DazzleKSampler] DEPRECATION: noise_seed=-2 is deprecated; latent_role=auto now infers dispatch from the latent dict shape regardless of seed. ...
    ```
  - The "auto-mode dispatched" informational warning from HV.1 should NOT appear (it's gated on `noise_seed != -2`).
- [ ] **Why this matters:** Existing v0.1.1 workflows must keep working. The deprecation message tells users how to migrate.

### HV.3 — `noise` role hides the seed widget

Validates: the JS extension reacts to role changes.

- [ ] **Steps:**
  1. Add a `DazzleKSampler` (or `DazzleSharkSampler` / `DazzleBongSampler` / `DazzleTauSampler`) to a workflow.
  2. Confirm both `latent_role` and `seed` are visible in the node's widget stack.
  3. Change `latent_role` to `noise` (or `noise+latent_image`).
  4. Observe the seed widget.
- [ ] **Expected:**
  - The seed widget disappears (collapses to zero height).
  - The node's overall height shrinks accordingly.
- [ ] **Steps (continued):**
  5. Change `latent_role` to `auto` (or `latent_image` or `seed_driven`).
- [ ] **Expected:**
  - The seed widget reappears.
- [ ] **Why this matters:** When `latent_role = noise` or `noise+latent_image`, the seed is unused (noise comes from upstream). Hiding it prevents the user from thinking the seed controls anything.

### HV.4 — Saved workflow restores seed-hidden state

Validates: the JS extension applies on workflow load.

- [ ] **Steps:**
  1. Build a workflow with a `DazzleKSampler` set to `latent_role = noise`. Confirm the seed widget is hidden.
  2. Save the workflow (Ctrl-S → Save / Save As).
  3. Refresh the page, or load the workflow JSON via "Load".
- [ ] **Expected:**
  - When the node is created on load, `latent_role` is set to `noise` and the seed widget is already hidden — no manual interaction needed.
- [ ] **Why this matters:** ComfyUI calls `nodeCreated` when loading saved workflows; the extension calls `syncSeedVisibility` at the end of `nodeCreated` so the saved-state nodes start correct.

### HV.5 — `seed_driven` ignores upstream `use_as_noise`

Validates: the explicit-override role takes precedence.

- [ ] **Steps:**
  1. Same Shape B-producing workflow as HV.1.
  2. Set DazzleKSampler `latent_role = seed_driven`, `seed = 42`.
  3. Queue.
- [ ] **Expected:**
  - Output looks like a standard txt2img with seed-driven noise (NOT influenced by upstream noise character).
  - Console:
    ```
    [DazzleKSampler] latent_role=seed_driven, detected=shape_b, applied=seed_driven, noise_seed=42
    [DazzleKSampler] WARNING: latent_role=seed_driven: ignoring upstream use_as_noise/noise (detected shape_b); generating noise from seed.
    ```
- [ ] **Why this matters:** This is the explicit "force seed-driven" escape hatch — users debugging a workflow should be able to bypass any upstream shaping.

### HV.6 — Explicit role mismatch produces a warning + safe fallback

Validates: the helper warns instead of crashing on mismatch.

- [ ] **Steps:**
  1. Workflow: stock `EmptyLatentImage` → `DazzleKSampler(latent_role=noise+latent_image, seed=42)` → Preview.
  2. Queue.
- [ ] **Expected:**
  - No crash. Output is a normal txt2img-from-empty result (Shape A fall-back).
  - Console:
    ```
    [DazzleKSampler] latent_role=noise+latent_image, detected=shape_a, applied=shape_a, noise_seed=42
    [DazzleKSampler] WARNING: latent_role=noise+latent_image expects Shape C ...; got shape_a. Falling back to Shape A: samples as init, noise generated from seed.
    ```
- [ ] **Why this matters:** Mismatch handling is a safety net — the user picked an explicit role, the upstream doesn't match, and we degrade gracefully with a clear console message rather than a tensor error.

### 4.2 `auto` + `seed=-2` + Shape A → DEPRECATION fires, no shape-dispatch

- [ ] **Why:** The deprecation log fires on `seed=-2 + role=auto` regardless of what's dispatched, so users see it consistently.

- [ ] Workflow: SmartResCalc Shape B → `DazzleKSampler(role=noise, seed=-2)` → Preview.
- [ ] **Console:** `applied=shape_b`. NO DEPRECATION line (the deprecation only fires on `role=auto` per the helper logic).
- [ ] **Why:** Explicit role means the user has migrated; no need to nag.

### 4.4 `auto` mode emits informational notice exactly once per generation

- [ ] Workflow: SmartResCalc Shape B → `DazzleKSampler(role=auto, seed=42, steps=20)` → Preview.
- [ ] **Console:** Only ONE `[DazzleKSampler] WARNING: auto-mode dispatched to Shape B ...` line per queued prompt — not per step. (The dispatch site gates `print()` on `total_steps_iter == 0`.)

## What the automated tests DON'T cover

- **Real ComfyUI sampling** — unit tests use the pure helper; they do not run a UNet, scheduler, or VAE. Whether dispatch produces *visibly* different compositions is human-verified only.
- **JS extension behavior** — the seed-widget hiding is browser-side; no automated test exercises it. HV.3, HV.4, and Section 3 cover this.
- **Console log wording** — unit tests don't capture stdout; visual verification of `latent_role=...`, WARNING, and DEPRECATION lines is human-only.
- **Workflow JSON load** — no automated test loads real workflow files; HV.4 and 3.4 cover this.
- **Cross-node uniformity** — automated tests target the helper, not the per-node `INPUT_TYPES`/`main()` wiring. Section 1 (presence sweep) is the human verification that the widget shipped on all 7 nodes.
- **Migration messaging clarity** — whether the DEPRECATION text is actually *useful* to users is a human judgment.

## Reset between sections

- **`latent_role`**: explicitly reset to `auto` between most tests.
