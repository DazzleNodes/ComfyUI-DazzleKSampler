# Human Test Checklist: Seed Widget Recovery + Determinism Docs (v0.1.3-alpha)

**Target version:** v0.1.3-alpha
**Ships with commit:** TBD (`fix: seed widget #12, correct latent_role docs, expose noise_type widgets`)
**Companion automated tests:** 36 tests in `tests/test_latent_noise_protocol.py` (35 existing + 1 new seed-plumbing regression test).

## What this is

A human-runnable companion to the v0.1.3-alpha release. v0.1.3 ships three
related changes that need real-ComfyUI verification:

1. **Bug fix [#12](https://github.com/DazzleNodes/ComfyUI-DazzleKSampler/issues/12)** —
   Seed widget no longer disappears after wiring an input to the seed socket
   and changing other widget values. Fixed by removing the conditional
   seed-hiding JS extension that was based on a wrong premise (the seed *is*
   used by per-step noise machinery regardless of `latent_role`).
2. **Docs/UX correction** — Tooltips, README, wiki, and CHANGELOG language
   for v0.1.2-alpha incorrectly stated the seed was unused under
   `latent_role ∈ {noise, noise+latent_image}`. Corrected to accurately
   reflect that `latent_role` only controls the *initial* noise tensor at
   sigma_max; the seed always drives per-step ancestral/SDE injection.
3. **New: structured first-step noise-source banner** that prints the
   active noise-source configuration at the start of every generation, so
   the seed/noise-source state is self-documenting at runtime.
4. **New: `noise_type_init`, `noise_type_sde`, `noise_type_sde_substep`
   widgets** on DazzleKSampler ([#13](https://github.com/DazzleNodes/ComfyUI-DazzleKSampler/issues/13)
   Phase 1+2). Lets users select the spectral content of initial-noise
   and per-step noise budgets independently. All default to `gaussian`
   (current behavior preserved).

Run this before tagging `v0.1.3-alpha`.

## Prerequisites

1. **Working ComfyUI install** with `dazzle-ksampler` and
   `smart-resolution-calc` custom nodes loaded.
2. **A checkpoint** for sampling (any model — these tests are model-agnostic).
3. **An upstream image** (any 512×768 or 1024×1024 portrait) for SmartResCalc
   `img2noise` mode.
4. **Console access** — keep ComfyUI's terminal visible. The dispatch site
   now logs:
   ```
   [DazzleKSampler] latent_role=noise, detected=shape_b, applied=shape_b, noise_seed=5225
   [DazzleKSampler] noise sources: initial=upstream tensor (latent_role=noise);
                    per-step=seed 5225 (deterministic). Set eta=0 to make
                    output fully determined by upstream noise.
   ```
5. **Reset state between sections:**
   - Right-click each Dazzle sampler node and confirm the seed widget
     is visible.
   - Set `latent_role = auto` between most tests; switch explicitly per section.
   - Restart ComfyUI between major test groups if widget state seems stale.

## High-Value Verification (~10 minutes)

If you only have 10 minutes, run these in order.

### HV.1 — Bug #12 regression: seed widget survives wire+widget-change

Validates: the v0.1.2-alpha bug where seed widget vanished after wiring an
input + changing another widget is fixed in v0.1.3.

- [ ] **Steps:**
  1. Build: `SmartResCalc → DazzleKSampler → VAE Decode → Preview`.
  2. Wire SmartResCalc's `seed` output to DazzleKSampler's `seed` input.
     - The seed widget on DazzleKSampler should now show as a "converted
       widget" (input slot, no number field).
  3. **Without** disconnecting the wire, change DazzleKSampler's `denoise`
     from `1.0` to `0.5` (or any other widget value).
  4. Disconnect the wire from the seed input.
- [ ] **Expected:**
  - The seed widget reappears as a normal numeric widget after disconnection.
  - It is editable and accepts new values.
  - No console errors related to widget state.
- [ ] **Pre-v0.1.3 symptom (should NOT appear):**
  - Seed widget permanently vanished, no number field, only an input socket
    that does nothing. Recovery required deleting and recreating the node.

### HV.2 — First-step noise-source banner: 5 cases

Validates: the new structured banner reports the correct configuration for
each `latent_role` × `noise_seed` combination. Watch the console after each
queue.

- [ ] **Case A — `latent_role=auto`, seed=42, Shape A (img2img) upstream:**
  Build: `EmptyLatentImage → DazzleKSampler`. Queue.
  Expected console line:
  ```
  [DazzleKSampler] noise sources: initial-noise+per-step driven by seed 42
                   (latent_role=auto, applied=shape_a).
  ```

- [ ] **Case A2 — `latent_role=auto`, seed=42, Shape B (latent-as-noise) upstream:**
  Build: `SmartResCalc(dimensions only, fill_type=DazNoise:Brown) → DazzleKSampler`
  with `latent_role=auto`, `seed=42`. Queue.
  Expected console line (note: `applied=shape_b`, not `shape_a` — this is the
  case that motivated routing the banner on `_disp.applied` rather than just
  `latent_role`):
  ```
  [DazzleKSampler] noise sources: initial=upstream tensor (latent_role=auto,
                   applied=shape_b); per-step=seed 42 (deterministic). Set eta=0
                   to make output fully determined by upstream noise.
  ```

- [ ] **Case B — `latent_role=latent_image`, seed=42:**
  Build: `LoadImage → VAE Encode → DazzleKSampler` with `latent_role=latent_image`. Queue.
  Expected console line:
  ```
  [DazzleKSampler] noise sources: initial-noise+per-step driven by seed 42
                   (latent_role=latent_image, applied=shape_a).
  ```

- [ ] **Case C — `latent_role=seed_driven`, seed=42:**
  Build: any latent → DazzleKSampler with `latent_role=seed_driven`. Queue.
  Expected console line:
  ```
  [DazzleKSampler] noise sources: initial=zeros; per-step + sample-noise=seed 42 (true txt2img).
  ```

- [ ] **Case D — `latent_role=noise`, seed=5225, Shape B upstream:**
  Build: `SmartResCalc(dimensions only, fill_type=DazNoise:Brown) → DazzleKSampler` with
  `latent_role=noise`, `seed=5225`. Queue.
  Expected console line:
  ```
  [DazzleKSampler] noise sources: initial=upstream tensor (latent_role=noise,
                   applied=shape_b); per-step=seed 5225 (deterministic). Set eta=0
                   to make output fully determined by upstream noise.
  ```

- [ ] **Case E — `latent_role=noise`, seed=-2:**
  Same workflow as case D but with `seed=-2`. Queue.
  Expected console line:
  ```
  [DazzleKSampler] noise sources: initial=upstream tensor (latent_role=noise,
                   applied=shape_b); per-step=torch.initial_seed()+1
                   (seed=-2 is non-deterministic across sessions).
  ```

### HV.3 — Determinism recipe: `eta=0` produces seed-independent output

Validates: with `eta=0` and `latent_role=noise`, KSampler seed is a dead input.
Empirical test that drove the v0.1.3 docs correction.

- [ ] **Steps:**
  1. Build: `SmartResCalc(dimensions only, fill_type=DazNoise:Plasma, seed=fixed) → DazzleKSampler → VAE Decode → SaveImage`.
  2. DazzleKSampler: `latent_role=noise`, `eta=0`.
  3. Generate three runs with seeds `5225`, `9999`, and `-2` (all with eta=0).
- [ ] **Expected:**
  - All three output images are **bit-identical** (or visually
    indistinguishable). The `[DazzleKSampler] noise_mean` and `noise_std`
    debug lines should also match.
- [ ] **Sanity contrast:** repeat the test with `eta=0.5` (default) — outputs
  should now differ between the three seeds (confirms the per-step path is
  what depends on the seed).

### HV.4 — `noise_type_*` widgets: defaults preserve prior behavior

Validates: adding the new noise-type widgets with `gaussian` defaults
produces output identical to v0.1.2-alpha.

- [ ] **Steps:**
  1. Build any standard workflow that worked in v0.1.2-alpha. Note the
     three new widgets at the bottom of the required block:
     `noise_type_init`, `noise_type_sde`, `noise_type_sde_substep`.
  2. Verify all three default to `gaussian`.
  3. Queue with a fixed seed. Save the output.
  4. **Optional regression check:** if you have a known v0.1.2-alpha output
     for the same workflow + seed, compare. Should be bit-identical.
- [ ] **Expected:**
  - Output matches what v0.1.2-alpha produced for the same configuration.
  - No console errors related to the new widgets.

### HV.5 — `noise_type_sde` non-gaussian visibly changes output

Validates: the new widget actually affects output (not a no-op).

- [ ] **Steps:**
  1. Same workflow as HV.4 with a fixed seed.
  2. Run once with `noise_type_sde=gaussian` (default). Save output.
  3. Run again with `noise_type_sde=brown` (or `pink` / `blue` / `pyramid_*`).
  4. Run again with `noise_type_sde=plasma` (if available in the dropdown).
- [ ] **Expected:**
  - Output differs between runs. Brown/pink runs should look smoother /
    more low-frequency-dominant; blue/violet runs should look grainier /
    more high-frequency-dominant; plasma should show structured patterns.
  - The exact magnitude of difference depends on `eta` — higher `eta`
    amplifies the spectral choice.

## What mocks don't cover (why this checklist exists)

The 36 automated tests in `tests/test_latent_noise_protocol.py` exercise the
pure helper `resolve_latent_as_noise()` with mock latent dicts. They cannot
verify:

- ComfyUI's widget→input conversion machinery (the actual #12 failure mode)
- Console output ordering and the new banner format under real ComfyUI logging
- Tooltip rendering in the ComfyUI frontend
- Visual output identity under `eta=0` (requires real diffusion samples)
- Spectral character of non-gaussian noise types (requires real diffusion samples)

Run this checklist before tagging the release.
