# Human Test Checklist: seed=-2 Dual-Role Bug Fix (Shape B init-zeroing)

**Target version:** v0.1.1-alpha
**Ships with commit:** TBD (`fix: zero init slot in Shape B latent-as-noise dispatch`)
**Companion automated tests:** 13 tests in `tests/test_latent_noise_protocol.py` (Shape A/B/C/D × seed combinations + protocol invariants)

## What this is

A human-runnable companion to the protocol unit tests for the `seed=-2` dual-role fix. The unit tests exercise the pure helper `resolve_latent_as_noise()` with mock latent dicts; this checklist exercises the full pipeline — SmartResCalc → DazzleKSampler → VAE Decode → Preview — under real ComfyUI with a real model. The whole point is to verify that when `fill_type` and `seed` change, the rendered composition actually varies. Mocks cannot tell you that; only your eyes can.

Run this before tagging `v0.1.1-alpha` and shipping the fix.

## Prerequisites

1. **Working ComfyUI install** with `dazzle-ksampler` and `smart-resolution-calc` custom nodes loaded.
2. **Any SDXL/Qwen/Wan checkpoint** loaded in a Load Checkpoint or UNETLoader node. The fix is model-agnostic — pick whatever you have.
3. **A test image** for img2img/img2noise paths (any 512×768 or 1024×1024 portrait works).
4. **Reference workflow JSON** (project-private):
   - `Smart-Res-Calc_testing_seed-and-latent-noise-outputs_with-ksampler.json` — minimal Shape B harness
5. **Console access** — keep ComfyUI's terminal visible. The fix prints distinguishable log lines:
   - `[ClownsharKSampler] Using input latent as noise — init zeroed (...)` — Shape B fix path
   - `[ClownsharKSampler] Using provided shaped noise (noise key, ...)` — Shape C preserved path
6. **Reset state between sections**:
   - Right-click DazzleKSampler → "Set noise_seed" to a known value before each test (or leave at `-2` for fix tests).
   - In SmartResCalc, toggle `fill_type` and `image_purpose` between tests as instructed.
   - Disconnect / reconnect upstream nodes per shape under test.

## High-Value Verification (~5 minutes)

**If you only have 5 minutes**, run these tests. They cover the highest-impact behavior the fix changes. If any fail, the fix is broken and the detailed sections below probably can't save you.

### HV.1 — Shape B + varying seeds produces visually different compositions

Validates: the fix actually does what it's supposed to do — break the dual-role anchor so seed variation shows up in output.

- [ ] **Steps:**
  1. Build / load: `SmartResCalc(image_purpose='dimensions only', fill_type='DazNoise: Brown', blend_strength=0.8) → DazzleKSampler → VAE Decode → Preview`. **No image input attached** to SmartResCalc (or if attached, image_purpose stays 'dimensions only').
  2. Set DazzleKSampler `noise_seed = -2`.
  3. Queue prompt. Note composition.
  4. Change SmartResCalc internal `seed` (the seed widget on SmartResCalc itself) to a different value (e.g., 12345 → 67890).
  5. Queue again. Compare.
- [ ] **Expected:**
  - Compositions are noticeably different — different camera angles, different subject placement, different background structure.
  - Console shows `[ClownsharKSampler] Using input latent as noise — init zeroed (Shape B)` for both runs.
- [ ] **Why this matters:** Pre-fix, the same tensor occupied both noise and init roles, so SmartResCalc's per-seed noise variation was overpowered by the init-anchor. If compositions look identical or near-identical across SmartResCalc seed changes, **the fix did not take effect**.

### HV.2 — Shape B + varying fill_type produces visually different compositions

Validates: upstream noise *character* (fill_type) now actually drives output.

- [ ] **Steps:**
  1. Same workflow as HV.1, with `noise_seed = -2`, SmartResCalc seed fixed (e.g., 12345).
  2. Queue with `fill_type = DazNoise: Brown`, blend_strength = 0.8.
  3. Change to `fill_type = DazNoise: Plasma`, queue.
  4. Change to `fill_type = DazNoise: Pink`, queue.
  5. Change to `fill_type = DazNoise: Gaussian`, queue.
- [ ] **Expected:**
  - Each fill_type produces a visibly different composition (different texture/structure character).
  - Pre-fix symptom would be: all four look almost identical. Post-fix: clear differences.
- [ ] **Why this matters:** This is the original user-reported regression. If fill_type changes don't move the composition meaningfully, the fix is not active or there's another anchor in the pipeline.

### HV.3 — Shape C (img2img + img2noise) preserves img2img behavior

Validates: the fix does NOT break the case where samples *should* be the init image.

- [ ] **Steps:**
  1. Workflow: `LoadImage → SmartResCalc(image_purpose='img2img + img2noise', fill_type='DazNoise: Brown', blend_strength=0.5, vae attached) → DazzleKSampler(noise_seed=-2, denoise=0.5) → VAE Decode → Preview`.
  2. Queue prompt.
  3. Inspect output.
- [ ] **Expected:**
  - Output preserves recognizable composition / subject from the input image (img2img is doing its job).
  - Console shows `[ClownsharKSampler] Using provided shaped noise (noise key, Shape C)` — NOT `init zeroed`.
- [ ] **Why this matters:** The regression guard. The fix gates init-zeroing on `if "noise" in latent_unbatch:` — Shape C must keep its `samples` as init. If the output looks like pure noise → image (no img2img anchoring) the gate is broken.

### HV.4 — fill_type=black falls through to seed-driven noise (Shape D)

Validates: trivial fills don't trigger the latent-as-noise branch.

- [ ] **Steps:**
  1. Workflow: `SmartResCalc(image_purpose='dimensions only', fill_type='black') → DazzleKSampler(noise_seed=-2) → VAE Decode → Preview`. No image attached.
  2. Queue twice with the same DazzleKSampler `noise_seed=-2` setting and the same SmartResCalc seed.
- [ ] **Expected:**
  - Two runs produce **bit-identical output** (or visually identical to inspection).
  - Console shows the standard noise generation log (NOT `init zeroed`, NOT `provided shaped noise`).
- [ ] **Why this matters:** `fill_type=black` produces Shape D (no `use_as_noise` flag). `seed=-2` falls through to deterministic seed-driven noise. Two runs with `-2` are deterministic relative to the underlying seed, so they must match. If they vary, the dispatch is wrong.

### HV.5 — Stock EmptyLatentImage works as latent_image input

Validates: the legacy ComfyUI path is unaffected.

- [ ] **Steps:**
  1. Workflow: `EmptyLatentImage(1024×1024) → DazzleKSampler(noise_seed=42) → VAE Decode → Preview`.
  2. Queue.
- [ ] **Expected:**
  - Standard txt2img output. No console errors. No mention of `init zeroed` or `Shape B/C` in logs.
- [ ] **Why this matters:** Confirms the helper short-circuits cleanly when `use_as_noise` is missing — EmptyLatentImage produces Shape D, must fall through.

### HV.6 — seed!=-2 with Shape B latent ignores the use_as_noise flag

Validates: the `-2` is the only opt-in.

- [ ] **Steps:**
  1. Same as HV.1 workflow (SmartResCalc producing Shape B with `use_as_noise=True`).
  2. Set DazzleKSampler `noise_seed = 12345` (any non-`-2` value).
  3. Queue.
- [ ] **Expected:**
  - Standard seed-driven generation. Console does NOT log `Using input latent as noise` or `Using provided shaped noise`.
  - Output reflects normal txt2img with the seeded noise generator (the upstream latent's structure is ignored).
- [ ] **Why this matters:** Documents that the fix is gated on `noise_seed == -2`. Users who want seed-driven generation must NOT have `-2` selected — and the fix must not accidentally activate for other seeds.

## Section 3: Upstream Graceful-Fallback (SmartResCalc Image Disconnect)

The user expressed concern about workflows where the image input on SmartResCalc was previously connected but is now disconnected, with `image_purpose` still set to `img2img`/`img2noise`/`image + noise`/`img2img + img2noise`. Verify SmartResCalc handles this gracefully.

### 3.1 Disconnect image with `image_purpose='img2img'` set

- [ ] **Steps:**
  1. Build: `LoadImage → SmartResCalc(image_purpose='img2img', vae attached) → DazzleKSampler → Preview`. Queue once.
  2. Disconnect the image link (LoadImage → SmartResCalc).
  3. Queue again.
- [ ] **Expected:**
  - SmartResCalc does NOT crash.
  - Either: produces Shape D / empty (fall through), OR clearly logs that image_purpose was downgraded to `dimensions only` because no image was attached.
- [ ] **Note:** This is graceful-fallback behavior in SmartResCalc, not the dazzle-ksampler fix. If it crashes, file as a separate SmartResCalc issue.

### 3.2 Disconnect image with `image_purpose='img2img + img2noise'` set

- [ ] **Steps:** Same as 3.1 with `image_purpose='img2img + img2noise'`.
- [ ] **Expected:** Same — graceful fallback, no crash.

## What the automated tests DON'T cover

- **Real ComfyUI sampling** — the unit tests use the pure helper; they do not actually run a UNet, scheduler, or VAE. Whether seed/fill_type variation produces *visibly* different compositions is human-verified only.
- **SmartResCalc upstream behavior** — the fix is dazzle-ksampler-side; SmartResCalc's per-seed and per-fill_type noise variation is independently verified.
- **Console log wording** — unit tests don't capture stdout; visual verification of the `Shape B`/`Shape C` log lines is human-only.
- **Workflow file load / GUI behavior** — no automated test loads a real workflow JSON.
- **Image-disconnect graceful fallback in SmartResCalc** — separate concern, separate node, separate test surface.
- **Performance** — does the fix add measurable overhead? (Should be zero — `torch.zeros_like` is one allocation that was already implicit in the previous arithmetic.)
- **Bit-identicality of Shape D `seed=-2` runs** — automated tests assert dispatch returns `not_applied`, but actual deterministic-noise generation downstream is not exercised.

## Reset between sections

Between the destructive / state-mutating sections:

- **DazzleKSampler `noise_seed`**: re-set to `-2` (or `42`, etc.) explicitly each section. ComfyUI sometimes auto-randomizes seeds depending on widget config.
- **SmartResCalc `image_purpose` and `fill_type`**: explicitly set per-test.
- **Image input link**: connect/disconnect per-test instructions.
- **Console**: clearing between major sections (Ctrl-L in most terminals) makes log inspection cleaner. Optional.
